# Security Policy

## Supported Versions

Security fixes and exploit patches are generally provided for the latest stable release of the mod.

| Version | Supported |
| ------- | --------- |
| Latest Release | ✅ |
| Previous Stable Release | ⚠️ Limited Support |
| Older Versions | ❌ |

> Development builds, alpha versions, and abandoned releases may not receive security fixes. If you find an exploit in an older version, please verify whether it exists in the latest release before reporting.

## Reporting a Vulnerability

Thank you for helping keep **Create: Project Stasis** fair and secure.

This project considers the following issues to be security-related:

- Item duplication exploits.
- Ways to bypass intended progression or restrictions.
- Unauthorized teleportation or movement exploits.
- Infinite resource generation.
- Exploits that allow players to bypass costs, limits, or permissions.
- Multiplayer exploits that allow one player to negatively affect another player's world.
- Any unintended mechanic that provides an advantage in survival gameplay.

### How to Report

For non-critical exploits:
- Please open a GitHub Discussion under the **Exploit Hunting** category.
- Include:
  - Mod version.
  - Minecraft version.
  - Steps to reproduce the issue.
  - Expected behavior.
  - Actual behavior.
  - Any other mods involved.

For serious exploits that could be widely abused on servers:
- Please contact the maintainer privately instead of posting the details publicly.
- Include enough information to reproduce the issue, but avoid sharing working exploit methods until a fix is available.

### What Happens After Reporting

After receiving a report:

1. The issue will be reviewed and reproduced if possible.
2. If confirmed, a fix will be prioritized based on severity.
3. Credit may be given to the person who discovered the exploit (unless they prefer to remain anonymous).
4. Once fixed, the issue may be documented in release notes or changelogs.

Please understand that response times may vary depending on severity, available time, and complexity of the fix.

Thank you for helping make Create: Project Stasis better for everyone!
